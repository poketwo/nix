# AGENTS.md

## Overview

NixOS and Kubernetes configuration for the **hfym-ds** cluster. This repo manages both the OS-level configuration (NixOS) for all nodes and the Kubernetes workload definitions, all declared in Nix.

## Repository Layout

### Two branches matter

| Branch | Contents | How it's updated |
|--------|----------|------------------|
| `main` | NixOS host configs + Kubernetes definitions in Nix | Manual commits |
| `cluster` | Rendered Kubernetes YAML manifests | Auto-generated by GitHub Actions from `main` |

**Never edit the `cluster` branch directly.** It is overwritten on every push to `main`.

### Directory structure (main branch)

```
hosts/           # Per-host NixOS configurations (one .nix per node)
hardware/        # Hardware-specific configs and presets (linode, gigabyte, supermicro, zfs)
modules/poketwo/ # Shared NixOS modules (networking, kubernetes, auth, nat64, etc.)
profiles/        # Base NixOS profile applied to all hosts
kubernetes/      # Kubernetes workload definitions (transpire Helm + raw resources)
  core/          #   Cluster infrastructure (cilium, argocd, cert-manager, metallb, etc.)
  monitoring/    #   Observability stack (prometheus, grafana, loki, mimir, alloy)
  poketwo/       #   Main application workloads
  guiduck/       #   GUIDuck application
  poketwo-staging/        # Staging environments
  poketwo-staging-private/
  outline/       #   Outline wiki + Retool
  extras/        #   Vault secrets, misc
  users/         #   Per-user resources
secrets/         # agenix-encrypted secrets (.age files)
```

### Key files

- `flake.nix` — Defines all inputs (nixpkgs, agenix, deploy-rs, transpire), host configs, and the `kubernetes` package output.
- `profiles/base.nix` — Base config for all hosts: nftables firewall, systemd-networkd, Cloudflare DNS, SSH, ZFS.
- `modules/poketwo/kubernetes.nix` — kubeadm setup, CRI-O, kubelet config, firewall ports.
- `modules/poketwo/network.nix` — Network bonding, IPv6 address assignment.
- `applicationset.yaml` — ArgoCD ApplicationSet that watches the `cluster` branch.

### Networking

- **IPv6-only** cluster. IPv4 is not enabled in Cilium.
- NAT64 (Jool) runs on hosts for workloads that need IPv4 connectivity (e.g. Discord API).
- Pod CIDRs are per-node subnets of the node's `/32` allocation (e.g. `2606:c2c0:5:1:131::/80` for glaceon).
- ndppd proxies NDP for pod CIDRs on the host's `inet0` interface so upstream routers can reach pod IPs.
- Cilium runs in native routing mode with kube-proxy replacement.
- MetalLB provides IPv6 load balancer addresses from `2606:c2c0:5:1:3::/112`.

## Deployment

### NixOS changes (hosts, modules, hardware)

Deploy to a specific host:
```bash
nixos-rebuild switch --target-host <host>.hfym.co --flake .#<host>
```

Or use deploy-rs:
```bash
deploy .#<host>
```

### Kubernetes changes

1. Edit files under `kubernetes/` on the `main` branch.
2. Commit and push to `main`.
3. GitHub Actions runs `nix build .#kubernetes`, renders YAML, and pushes to `cluster` branch.
4. ArgoCD auto-syncs from `cluster` branch.

To build manifests locally:
```bash
nix build .#kubernetes
ls result/    # rendered YAML per namespace
```

To apply manually when ArgoCD is down:
```bash
nix build .#kubernetes
kubectl apply -f result/<namespace>/
```

### Kubernetes manifest format (transpire)

Each `.nix` file under `kubernetes/` defines resources for a namespace. Helm charts use `transpire.fetchFromHelm`:

```nix
{ transpire, ... }:
{
  namespaces.<namespace> = {
    helmReleases.<name> = {
      chart = transpire.fetchFromHelm {
        repo = "https://...";
        name = "...";
        version = "...";
        sha256 = "...";
      };
      values = { ... };
    };

    # Raw resources
    resources."<apiVersion>".<Kind>.<name> = { ... };
  };
}
```

When upgrading a Helm chart, update `version` and `sha256`. To get the new hash, set `sha256` to an empty string (`""`), run `nix build .#kubernetes`, and copy the correct hash from the error message.

### Secrets

Secrets are managed with [agenix](https://github.com/ryantm/agenix). Encrypted `.age` files live in `secrets/`. Public keys for decryption are defined in `secrets/secrets.nix`.

Kubernetes secrets are managed via HashiCorp Vault + Vault Secrets Operator, not agenix.

## Important Considerations

- **IPv6 only.** Do not add IPv4-dependent configurations to Cilium or cluster networking.
- **Pod CIDR reachability** depends on ndppd running on each worker node. If pods on a node become unreachable from outside the node, check that ndppd is running and configured for the node's pod CIDR.
